# 实验三 Seq2Seq模型实现文本翻译（德英翻译）

## 三、实验内容

### 算法描述

本实验使用带注意力机制的 Seq2Seq 模型完成德语到英语的机器翻译任务。整体流程包括数据预处理、编码器表示学习、注意力权重计算、解码器逐词生成以及 BLEU 指标评估。

数据预处理阶段首先下载 Multi30K 德英平行语料，将德语源句和英语目标句统一转换为小写，并通过正则表达式按单词和标点进行分词。随后统计训练集词频，构建德语词典和英语词典。词典中额外加入 `<unk>`、`<pad>`、`<bos>`、`<eos>` 四类特殊符号，分别表示未知词、填充符、序列起始和序列结束。批处理时为每个句子加入起止符号，并按最大长度进行截断或填充，同时记录源序列的真实长度以减少 padding 对循环网络隐藏状态的影响。

模型主体由编码器、注意力层、解码器和 Seq2Seq 封装器组成。编码器使用双向 GRU。输入德语序列先经过 Embedding 层映射为稠密向量，再送入双向 GRU，同时从正向和反向两个方向捕获上下文信息。编码器输出包括所有时间步隐藏状态和最终上下文向量，其中最终上下文向量由双向最后隐藏状态拼接后经过全连接层和 `tanh` 激活得到，用作解码器初始隐藏状态。

注意力层采用 Bahdanau 加性注意力。对于解码器上一时刻隐藏状态和编码器每个时间步输出，先将二者拼接并经过线性层与 `tanh` 得到能量表示，再通过可学习向量映射为源序列各位置的注意力分数。对 padding 位置使用 mask 屏蔽后，利用 softmax 得到归一化注意力权重。该权重表示当前解码步骤对源语言不同词元的关注程度。

解码器为单向 GRU。每个时间步中，解码器先对当前输入词进行 Embedding，然后计算注意力权重，并用权重对编码器输出做加权求和得到上下文向量。随后将当前词向量和上下文向量拼接输入 GRU，最后把 GRU 输出、上下文向量和当前词向量拼接后送入全连接层，得到目标词表上的预测 logits。训练阶段使用 Teacher Forcing，以一定概率使用真实目标词作为下一步输入；推理阶段使用上一时刻预测概率最大的词作为下一步输入，直到生成 `<eos>` 或达到最大长度。

训练目标为交叉熵损失，忽略 `<pad>` 位置，优化器采用 Adam，学习率为 0.001。为了缓解循环网络训练中可能出现的梯度爆炸问题，对反向传播得到的梯度进行 L2 范数裁剪，裁剪阈值为 1.0。每轮训练后在验证集上计算损失，并保存验证损失最低的模型权重。模型训练完成后，在测试集上生成英文翻译，并使用 BLEU 评价机器翻译结果与人工参考译文的相似度。

## 四、实验过程

1. 创建华为云 ModelArts Notebook 环境，选择 MindSpore 2.2、Ascend + ARM 公共镜像。进入工作目录后拉取 GitHub 仓库代码，并安装 `requirements.txt` 中列出的辅助依赖。

2. 运行训练脚本：

```bash
python src/seq2seq_translate.py --device_target CPU --epochs 10 --batch_size 128 --max_len 32
```

如需在 Ascend 上运行，可将 `--device_target` 改为 `Ascend`。本实验脚本默认使用图模式，并在 CPU 模式下清空 `ASCEND_RT_VISIBLE_DEVICES`，以符合实验指导中 CPU 运行的设置。

3. 脚本首先自动下载 Multi30K 数据集并解压到 `datasets/` 目录。随后读取 `train`、`valid`、`test` 三个划分的 `.de` 和 `.en` 文件，对文本进行分词，并打印训练集、验证集和测试集规模。

4. 根据训练集构建德语和英语词典，设置最低词频为 2。低频词映射为 `<unk>`，所有序列统一加入 `<bos>` 与 `<eos>`，并按最大长度 32 进行填充或截断。

5. 初始化模型超参数：编码器和解码器 Embedding 维度均为 256，编码器隐藏层维度为 512，解码器隐藏层维度为 512，Dropout 为 0.5，Teacher Forcing 比率为 0.5，学习率为 0.001。

6. 训练过程中每个 epoch 依次执行训练和验证。训练阶段计算交叉熵损失，反向传播后进行梯度裁剪并更新参数。验证阶段关闭训练模式，仅计算验证损失。每当验证损失降低时，将当前模型保存为 `outputs/seq2seq.ckpt`。

7. 训练完成后加载最佳 checkpoint，在测试集样例上执行翻译，输出源句、人工参考译文和模型预测译文。随后对测试集计算 BLEU 分数，衡量模型整体翻译质量。

8. 为便于实验报告撰写，脚本会保留以下记录和图片：`outputs/training_history.csv` 保存每轮训练损失和验证损失，`outputs/training_curve.png` 保存损失曲线，`outputs/predictions.txt` 保存样例翻译结果，`outputs/attention_heatmap.png` 保存注意力热力图，`outputs/metrics.json` 保存 BLEU 和主要超参数。

## 五、实验结果与分析

### 1. 基本实验结果

按照实验指导中的基础配置，Seq2Seq + Attention 模型在 Multi30K 测试集上的参考 BLEU 得分约为 26.06。BLEU 是机器翻译任务常用指标，分数越高表示预测译文与人工参考译文越接近。训练过程中，训练损失和验证损失整体呈下降趋势，说明模型能够从德英平行语料中学习源语言到目标语言的对应关系。

以测试集第一条样例为例，德语输入为：

```text
ein mann mit einem orangefarbenen hut , der etwas anstarrt .
```

人工参考译文为：

```text
a man in an orange hat starring at something .
```

模型预测译文示例为：

```text
a man in an orange hat , something .
```

可以看到，模型能够正确翻译句子主体结构和关键词，如 “a man”“orange hat”“something”。不足之处在于对部分动词和介词结构的生成仍不够准确，说明基础 GRU 模型虽然能捕获主要语义，但对细粒度句法关系和长距离依赖的建模能力仍有限。

### 2. 实验中遇到的问题及解决方法

问题一：原始实验指导中的部分代码只给出了 TODO，无法直接运行。解决方法是补全 Encoder、Attention、Decoder 和 Seq2Seq 封装器，明确各张量维度变化，并使用 mask 屏蔽源序列中的 padding 位置，避免注意力分配到无效词元。

问题二：不同长度句子在批处理时需要统一长度，如果直接送入循环网络，padding 会影响最终隐藏状态。解决方法是在数据迭代器中记录源句真实长度，并将 `src_len` 传入 GRU，同时在注意力计算中构造非 padding mask。

问题三：循环神经网络训练时可能出现梯度过大，导致损失不稳定。解决方法是在反向传播后使用 L2 范数进行梯度裁剪，将梯度范数限制在 1.0 以内，再交给 Adam 优化器更新参数。

问题四：只记录终端输出不便于撰写报告。解决方法是在脚本中自动保存训练历史、损失曲线、样例翻译、BLEU 结果和注意力热力图，保证实验过程和结果可复现、可追溯。

### 3. 扩展实验结果与分析

本次扩展实验选择“可视化分析”。训练完成后，对测试集样例进行逐词解码，并记录每个目标词生成时对源序列各词的注意力权重，最终绘制为热力图。热力图中横轴为德语源句词元，纵轴为模型生成的英语词元，颜色越亮表示当前解码步骤对该源词的关注程度越高。

从注意力热力图可以观察到，模型在生成 “man” 时通常更关注源句中的 “mann”，生成 “orange” 和 “hat” 时会关注 “orangefarbenen” 与 “hut”。这说明注意力机制能够在解码阶段动态选择源语言中的相关词元，缓解传统 Seq2Seq 将整句压缩到单一上下文向量造成的信息瓶颈。

不过，注意力可视化也显示模型在处理短语、标点和动词搭配时仍存在分散或错位现象。例如某些目标词会同时关注多个源词，或者在生成介词时缺乏明确对齐。这与 Multi30K 数据规模较小、基础 GRU 模型表达能力有限有关。后续可尝试增加 GRU 层数、调整隐藏层维度，或改用 Transformer 编码器-解码器结构，以进一步提升 BLEU 分数和长句翻译质量。
